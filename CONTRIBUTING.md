# Contributing to KnowTheScam

Thank you for contributing to the KnowTheScam scam database.

This project aims to build a structured, globally relevant, and technically consistent repository of documented scams.

---

## Contribution Rules

### 1. Follow the Official Template

All scam entries must:
- Use the approved scam-template.md
- Include all required metadata fields
- Follow folder-based category structure
- Maintain consistent scoring logic

Incomplete submissions will not be accepted.

---

### 2. No Live Malicious Infrastructure

Do NOT include:
- Active scam domains
- Cryptocurrency wallet addresses
- Personal identifying information
- Direct operational details that could enable abuse

Keep entries educational and analytical.

---

### 3. One Primary Category Rule

Each scam must:
- Live in exactly one folder
- Match the `category:` metadata field

Secondary traits should be listed under `tags`.

---

### 4. Severity Scoring Consistency

Severity score must equal:

financial_impact +
psychological_impact +
scale_of_operation +
technical_sophistication

Scores must be justified and proportional.

---

### 5. Source Quality

Cite reputable sources when possible:
- Government agencies
- Law enforcement reports
- Academic research
- Cybersecurity organizations

Avoid blogspam or unverified anecdotes.

---

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Add or modify scam entry
4. Ensure formatting is correct
5. Submit a pull request with a clear explanation

All submissions will be reviewed for structure and consistency.

---

## Code of Conduct

Contributors must maintain:
- Professional tone
- Neutral language
- No political framing
- No discriminatory or inflammatory content

This is a neutral educational resource.

---

Thank you for helping build a global scam database.
